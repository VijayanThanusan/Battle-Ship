21  FONCTIONNEMENT DE LA CLASSE GAME :
22 
23  La classe Game est la classe principale du jeu, elle regroupe les principale fonctions qui vont faire tourner le jeu.
24  Il y a principalement trois fonctions qui s'y trouve :
25  - Mode Solo
26  - Mode Joueur contre Joueur
27  - Mode Joueur contre IA
28 
29  1) Le mode solo nous permet de joueur seul : on choisi au départ une difficultée parmis trois difficultée,
30  en fonction de la difficultée choisi, le joueur devra terminer la partie en un certain nombre de coup limité
31  2) Le mode Joueur contre Joueur nous permet de jouer contre un autre joueur humain : chacun joue a sont tour jusqu'a
32  ce que l'un des joueurs ai trouvé tous les bato de l'adversaire.
33  3) Le mode Joueur contre IA nous permet de jouer contre l'ordinateur : le joueur commence a jouer, suivi de l'IA
34  si le joueur gagne alors sont tour continue et il reclic encore juska ce qu'il perd, il en est de même pour l'IA ..
35 
36  Ces trois modes sont au départs tous chargé dans la classe Acceuil, et lorsque l'on clique sur notre choix dans l'acceuil, il nous
37  redirige directement sur la scene en question
38  Pour ce faire il y a donc trois scènes de type "Scene" et trois groupe de type "Group" car il y a trois modes diffférent
39  Chaque mode est censé avoir une contenu différent donc on ne peut pas afficher les trois modes de jeu dans une seule scene et
40  dans un seul groupe.
41  Chaque groupe contient donc les éléments graphiques correspondant a son mode de jeu réspéctif.
42 
43  Chacune de ses trois fonctions possèdent leur variables réspéctives qui sont déclaré en tant qu'attributs dans la classe Game,
44  il sont déclaré en tant qu'attribut et non déclaré dans leur propre fonction car, ces variables sont en mode static certaines
45  d'entre elles seront utilisé mais dans d'autre classe, tels que dans la classe Case par exemple ou on affiche le score de la
46  grille a chaque clik sur une case.
47  Parmis ses variables on trouve principalement les grilles de chaque mode ; en tout 5 grille sont initialisée :
48  - La grille "joueur_solo" pour le mode solo
49  - Deux grilles "joueur1" et "joueur2" pour les deux joueur du mode "Joueur contre Joueur"
50  - Une grille "joueur_mode_IA" pour la grille du joueur dans le mode "Joueur contre IA"
51  - Et enfin une variable de type IA dans la quelle la grille est déja initialisé dans la classe elle même.
52  Plusieurs grille ont été créé et non une seule ou deux pour évité les erreurs de contenu, et pour bien différencier chaque grille ...
53  cela est certes un peu plus encombrant question mémoire, mais plus compréhensible pour le code, on différencie bien de la sorte
54  quelle grille appartenant à quel mode de jeu ...
55 
56  Enfin une fonction reset permet de réinitialisé la grille passé en paramètre et y replace des bato, afin de pouvoir rejouer.
57 */
